Hospital362 Demo Tutorial
========================

This tutorial matches the official use cases. Each scenario can be performed using the application's menu or command interface.

---

Use Case 1: Dispense Prescribed Medication
------------------------------------------
Actor: Pharmacist
Precondition: A doctor has recorded an active prescription for an admitted patient.

Demo Steps:
1. Log in as Pharmacist (Tom Bradley in employees.txt).
2. Select the option to view pending patient prescriptions.
3. The system displays all unfulfilled prescriptions (e.g., RX-1002 for James Carter).
4. Select a prescription to process (e.g., RX-1002 for James Carter).
5. The system displays medication name, dosage, and required quantity.
6. Confirm you have enough inventory (see pharmacy_inventory.txt).
	- If not enough, cancel fulfillment and initiate a supply order (alternate flow).
	- If prescription is expired, system will warn and you must abort (alternate flow).
7. If all is valid, confirm fulfillment.
8. The system deducts the quantity from inventory and marks the prescription as fulfilled.

---

Use Case 2: Admit a New Patient
-------------------------------
Actor: Receptionist
Precondition: A patient walks into the hospital needing assistance.

Demo Steps:
1. Select "Admit Patient" in the application.
2. Enter new patient details (Name, Insurance, Injury, etc.).
3. If rooms are available (see rooms.txt), assign a room and confirm admission.
	- If no rooms are available, system will prompt to transfer patient (alternate flow).
4. Patient is added to the system and sent to the waiting area.
	- For emergencies, use the "Emergency Admit" option (alternate flow).

---

Use Case 3: Hire New Employee
----------------------------
Actor: Human Resources
Precondition: Worker applies for a hospital role.

Demo Steps:
1. Select "Register Employee" in the application.
2. Enter applicant details (e.g., Emily Turner or a new applicant).
3. Move applicant to interview stage and schedule interview.
4. Record interview outcome.
	- If not a good fit, send rejection (alternate flow).
5. If passed, perform background check.
	- If failed, send rejection and add to blacklisted_applicants.txt (alternate flow).
6. If passed, send job offer and begin onboarding.

---

Use Case 4: Discharge Existing Patient
--------------------------------------
Actor: Receptionist, BillingDepartment
Precondition: Patient is ready for release or has signed a Refusal of Treatment form.

Demo Steps:
1. Select "Discharge Patient" in the application.
2. Look up patient by name or ID (e.g., James Carter, P1002).
3. Confirm discharge authorization or refusal of treatment.
	- If refusal, request liability waiver (alternate flow).
4. Request bill from BillingDepartment.
5. Provide bill to patient and process payment or insurance claim.
6. Update patient status to Discharged.
7. Patient leaves hospital.

---

Use Case 5: Request Room Cleaning
---------------------------------
Actor: Nurse, Facilities Management Staff
Precondition: Room was recently used.

Demo Steps:
1. Select "Request Cleaning" in the application.
2. Choose a room (e.g., 400 or 112 from rooms.txt).
3. Select cleaning type (Standard, Deep, Biohazard, Maintenance).
	- For biohazard or maintenance, add notes as prompted (alternate flows).
4. Request is queued and assigned to Facilities staff (e.g., Marco Rivera).
5. Staff updates room status to clean after completion.

---

Tips:
- All demo data is pre-populated in the data/ directory.
- You can view or edit the .txt files for more details.
- For any scenario, follow the prompts in the application UI.
